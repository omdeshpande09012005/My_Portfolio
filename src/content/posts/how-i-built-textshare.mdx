## Building the Database Schema

I used Prisma migrations to version the database:

```prisma
model Paste {
  id        String   @id @default(cuid())
  content   String
  createdAt DateTime @default(now())
  expiresAt DateTime
  password  String?
  viewCount Int      @default(0)
}

model Share {
  id        String   @id @default(cuid())
  type      String   // 'text', 'file', 'url', 'qr'
  data      Json
  expiresAt DateTime
  views     View[]
}
```

## Performance Optimizations

1. **Database Indexing**: Added indexes on frequently queried fields
2. **API Caching**: Implemented Redis-like caching patterns
3. **File Streaming**: Direct streaming for file downloads
4. **CDN Integration**: Vercel's edge network for static assets
5. **Database Connection Pooling**: Managed with Prisma client

## Deployment to Vercel

Deployment was seamless:

1. Connected GitHub repository to Vercel
2. Set environment variables (DATABASE_URL, API keys)
3. Configured build settings
4. One-click deployment

Vercel's serverless functions handle all API requests, PostgreSQL on managed hosting (Supabase) stores data, and the edge network serves static content.

## Key Lessons Learned

1. **Rate Limiting is Critical**: Implemented request throttling to prevent abuse
2. **Data Expiry Management**: Scheduled cleanup jobs to remove expired shares
3. **User Privacy**: Implement strict CORS and CSRF protections
4. **Monitoring**: Set up logging and error tracking with Sentry
5. **Documentation**: Write clear API docs for third-party integrations
